# Plan: RSS Feed for Reviews

**Date:** 2026-05-08
**Slug:** rss-feed

## Goal

`/reviews/rss.xml` serves a valid RSS feed listing all reviews sorted by date
descending, so readers can subscribe to new review notifications.

## Affected files

| Action | File |
|--------|------|
| Install | `@astrojs/rss` npm package |
| Create  | `src/pages/reviews/rss.xml.ts` |
| Create  | `docs/plans/2026-05-08-rss-feed.md` (this file) |

No changes required to `astro.config.mjs` — `site` is already set to
`https://cosmiccoding.com.au`.

## Context

- Reviews content collection is in `content/reviews/<slug>/index.md`
- Schema has: `title`, `auth`, `name`, `sentence`, `date`, `review`, `tags`, `links`
- `site` in `astro.config.mjs` = `'https://cosmiccoding.com.au'`
- Review pages are served at `/reviews/<slug>/`
- `@astrojs/rss` is not yet in `package.json`

## Tasks

1. `npm install @astrojs/rss` — add the RSS package
2. Create `src/pages/reviews/rss.xml.ts`:
   - GET endpoint using `@astrojs/rss`
   - `getCollection('reviews')`, sort by `date` descending
   - Map each entry to `{ title, description: sentence, author: auth, pubDate: date, link }`
   - Feed title: "Cosmic Coding — Book Reviews"
   - Feed description: "Progression fantasy and LitRPG book reviews"
   - customData includes `<language>en-us</language>`

## Verification

```bash
astro check && npm run build
```

Check that `dist/reviews/rss.xml` exists and contains valid XML with `<item>` entries.

No browser-tester visual review needed — this is a machine-readable XML endpoint.

## ADR

None — using the standard Astro RSS pattern with the official integration.
